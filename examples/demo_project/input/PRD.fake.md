# 示例医疗 B 端系统｜价格审批模块

## 项目背景与目标

当前价格申请和审批流程需要从线下沟通切换为线上闭环管理，实现申请、审批、执行反馈和历史追溯。

## 用户角色与权限

- 申请人：新增申请、查看列表、查看详情、执行反馈。
- 审批人：查看待审批、单条审批、批量审批、填写审批意见。
- 抄送人：只读查看流程结果和消息通知。
- 管理员：维护基础数据和系统配置。

## 核心业务流程

申请人提交申请 → 系统判断审批规则 → 审批人处理 → 审批通过或拒绝 → 申请人执行反馈 → 流程归档。

## 审批触发规则

当申请价格低于示例阈值，或影响示例挂网价时进入人工审批；否则系统自动通过。

## 页面说明

- PC 页面：申请列表、申请详情、审批中心、批量审批、最低价查询、消息中心。
- 移动端页面：发起申请、我的申请、申请详情、审批进度、执行反馈、消息详情。

## 待确认问题

- 移动端是否支持被拒后重新提交。
- 延期按钮展示条件和可延期次数。
